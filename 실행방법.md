# YJHWell 실행 방법 (오프라인 · Python)

2D 영구자석 모터 설계/해석 프로그램. **인터넷 없이** 로컬 Python으로 UI를 띄워
작업할 수 있습니다. (의존성 설치 때 한 번만 인터넷이 필요하고, 그 뒤로는 완전 오프라인.)

---

## 0. 준비물
- Windows + **Python 3.12** (3.13/3.14는 일부 과학 라이브러리 휠이 없을 수 있어 3.12 권장)
- 확인:
  ```
  py -3.12 --version
  ```
  안 깔려 있으면 https://www.python.org/downloads/ 에서 3.12 설치
  (설치 시 "Add python.exe to PATH" 체크).

---

## 1. 가장 쉬운 방법 (배치 파일, 더블클릭)
폴더 안에서:
1. **`setup.bat`** 더블클릭 — 가상환경(.venv) 생성 + 의존성 설치 (인터넷 1회 필요)
2. **`run.bat`** 더블클릭 — UI 실행 (이후 오프라인)

---

## 2. Python 명령으로 직접 실행 (수동)
폴더(예: `C:\Users\user\Desktop\YJHWELL`)에서 PowerShell/명령프롬프트를 열고:

```bat
:: (1) 가상환경 생성 — 최초 1회
py -3.12 -m venv .venv

:: (2) 의존성 설치 — 최초 1회 (인터넷 필요)
.venv\Scripts\python -m pip install --upgrade pip
.venv\Scripts\python -m pip install -r requirements.txt

:: (3) 실행 — 이후 언제나 오프라인 가능
.venv\Scripts\pythonw run.py      :: 창만 (콘솔 로그 없음)
.venv\Scripts\python  run.py      :: 콘솔 로그도 같이 보기
```

설치되는 패키지(고정 버전, `requirements.txt`): numpy, scipy, shapely, triangle,
pyqtgraph, pillow, PyQt6.

---

## 3. 프로그램 사용 흐름 (UI)
1. 리본 **Draw/Model** 탭에서 형상 작도 — 또는 메뉴로 **샘플 모터 삽입**(10극12슬롯)
   - **좌표계(Coordinates)**: `Relative CS`로 오프셋·회전된 작업좌표계를 만들면, 이후
     입력하는 좌표(좌표바·치수 다이얼로그)가 그 CS 기준으로 배치됩니다. `Set Global`로 복귀.
     (Maxwell의 Relative CS Offset/Rotate와 동일 — 캔버스에 빨강X/초록Y 트라이어드 표시)
   - **선택 모드(Maxwell식 단축키)**: 캔버스에서 **O**=객체 / **E**=모서리 / **V**=꼭짓점 /
     **F**=면 선택. (리본 Select 그룹 버튼으로도 가능) 용례:
     · **E**(모서리) 선택 후 *Assign Boundary(Vector Potential)* → 그 모서리에 경계 적용
     · **V**(꼭짓점) 선택 후 *Edit ▸ Fillet* → 그 코너를 라운딩(기본 반경 = MagnetR 변수)
     · **F**(면)/E(모서리) 선택 후 *Coordinates ▸ Face CS* → 그 위치/방향으로 좌표계 생성
2. 재질 할당 (Select Definition / View·Edit Material, B-H·철손 곡선)
3. **여자·경계·메시 설정** (객체 우클릭)
   - **Assign Excitation ▸**: **Coil**(Base Name `Ph_A` / Conductors `Zc` / Polarity ± →
     **Add Terminals** Ph_A_1~4·Re_Ph_A_1~4) · **Add Winding**(PhaseA/B/C, 전류식
     `Ia_max*sin(omega*time±120deg)`, Stranded, 병렬분기 a) · **Set Eddy Effects**(자석 ON) ·
     **Set Core Loss**(스테이터 ON; 재질에 철손계수 있을 때만 적용)
   - **Assign Boundary ▸ Vector Potential**(E로 모서리 선택), **Assign Band**(회전 Motion, init_pos)
   - **Assign Mesh Operation ▸ Inside Selection ▸ Length Based**: 객체별 **요소길이 세분화**
     (Band/Core/Magnet/Coil/Region — Apply to Initial/Enable/최대요소길이/추가요소수). 메시 생성 시 자동 적용.
4. **Simulation** 탭 → Solve Setup → **Analyze** (B-field 오버레이)
   - **Nonlinear (BH)** 토글을 켜면 철심 포화를 반영한 비선형 해석
   - **Solve Setup**: General 탭(Stop time/Time step) + **Save Fields 탭**
     (None / Every / **Custom**: Linear Step · Linear Count · Single Point · Single Point Sweep)
   - **Maxwell 2D ▸ Design Settings**: Material Thresholds + Symmetry Multiplier(대칭배수) +
     Model Depth(적층길이 L_stk) + **Skew Model**(슬라이스 수/스큐각)
   - Results 트리 **우클릭 → Create Transient Report ▸ New Trace…**(Category Torque/Winding/Loss →
     Quantity → Function) 또는 바로 Torque / Winding(Induced Voltage) / Transient / Core Loss
   - **Edit Color Map…**(B legend 더블클릭과 동일: Spectrum/Scale/Num.Division), **Clean Up Solutions…**
   - 결과 차트 **우클릭 → Average / Pk-Pk / Max / Min / RMS** 마커선 표시
5. **Results** 탭:
   - **Torque Plot** — 무여자 코깅 토크 vs 회전각
   - **Back-EMF** — 무부하 역기전력 (3상 InducedVoltage A/B/C)
   - **Load Torque** — 정격 3상 전류 부하 토크
   - **Transient EMF** — 시간영역 역기전력 파형 (전기 2주기, vs 시간[ms])
   - **Core Loss** — 철손(Steinmetz/Bertotti: 히스테리시스/와전류/과잉) [W]
   - **Field Overlay** — 자속밀도 컬러 컨투어
   - **Param Sweep (Optimetrics)** — 설계변수 1개 스윕 → 토크/Bmax/Back-EMF/부하토크
     출력 표 + CSV 내보내기
   - **Field Plot 양**: Mag_B/B_Vector/Mag_H/Jz/Core_Loss/Total_Loss 등 18종 + AllObjects 선택
   - **다중 디자인**: 프로젝트 트리 우클릭 **Insert/Copy Design**, 디자인 더블클릭으로 활성 전환
     (NoLoad/RatedLoad처럼 여러 디자인 공존; 풀린 디자인은 필드·메시까지 독립 복제됨)
   - **Set View Context**: 캔버스 **Time= 오버레이 더블클릭** → 솔루션/시간/속도·위치 표시 설정

> 극수·슬롯·속도·전류·turns·형상이 전부 변수라 **400W 한 종류가 아니라 다양한 모터**를
> 설계/해석할 수 있습니다.

## 3-1. FEMM 하이브리드 (선택 — 고정밀/검증)
내장 솔버는 실제 2D FEM이지만 1차요소·준정적이라 절대정밀도는 lite입니다. 검증된
오픈소스 **FEMM**(https://www.femm.info, 예: `C:\femm42`)이 설치돼 있으면:
```
.venv\Scripts\python -m pip install -r requirements-femm.txt   # pyFEMM(1회)
.venv\Scripts\python feedback\femm_xval.py                      # 내장 vs FEMM 교차검증
```
프로젝트 이름은 Project Manager 트리의 최상위 노드 **우클릭 → Rename Project…** 로 변경합니다.

## 4. 솔버 검증 (선택)
계산이 물리적으로 맞는지 자동 검증하는 하니스가 `feedback/`에 있습니다.
feedback-runner 스킬의 runner로 실행:
```
python <feedback-runner경로>\scripts\runner.py --config feedback\feedback.config.json
```
종료코드 0 = 모든 지표 허용오차 내(PASS). 설계무관 물리검증(3상 평형, e=-dλ/dt
자기일관성, dq vs FEM 토크, T∝I 선형성, 비선형 선형폴백 항등) + 회귀 베이스라인.

## 5. 문제 해결
- **`py`가 없다고 나옴** → `python -m venv .venv` 로 대체 (단, 기본 python이 3.12여야 함)
- **PyQt6 설치 실패 / 휠 없음** → Python이 3.12인지 확인 (`.venv\Scripts\python --version`)
- **`triangle` 빌드 오류** → 3.12 사용 시 미리 빌드된 휠이 설치됨. 3.13+에서 종종 실패.
- **창이 안 뜸** → `.venv\Scripts\python run.py` 로 실행해 콘솔 오류 메시지 확인
- **한글 경로/콘솔 깨짐** → 기능엔 영향 없음 (UI는 정상). 로그만 영향.
