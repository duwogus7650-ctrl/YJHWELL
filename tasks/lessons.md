- Korean 파일명을 Windows PowerShell 5.1에서 네이티브 exe(ffmpeg 등)에 넘기면 인코딩 깨짐("Illegal byte sequence") → ASCII 하드링크로 우회. (왜: WinPS 5.1이 레거시 코드페이지로 인자 전달)

- "라이트 버전"=경량(lightweight)이지 밝은 테마가 아님. 밝은 색 UI가 사용자에게 "밝은버전?" 혼동을 줌 → 단계적 MVP/테마 의도를 명확히 알리고, 모호어는 초기에 확인. (왜: 진척·의도 오해 방지)
- GUI 검증은 포그라운드 전환(SetForegroundWindow)이 거부될 수 있으니 QWidget.grab() 또는 PrintWindow(flag=2)로 캡처. (왜: 다른 창이 겹쳐도 결정적 캡처)
- Windows 콘솔(cp949)에서 print에 em-dash(—) 등 유니코드 있으면 UnicodeEncodeError → 로그 출력은 ASCII로 하거나 파일로.
- "영상과 똑같이" 요청: ① 기능을 임의로 '라이트'로 빼지 말 것(영상에 있으면 구현 대상) ② 내 테마/스타일 입히지 말고 원본 외형(여기선 AEDT 라이트 회색) 따를 것. 두 실수가 "영상과 다르다"의 근본 원인. (왜: 사용자 핵심 의도는 '동일 재현'이지 내 디자인이 아님)
- 막연한 "아직 아니야" 피드백은 추측 패치로 이어져 드리프트. 각 수정은 **해당 실제 프레임을 직접 열어 그 화면을 근거로** 재현할 것(예: CreateCircle 대화상자 = uniq_v1/u_0031). (왜: 추측 대신 근거 기반이라야 수렴)
- **검증을 특정 설계(400W)에 묶지 말 것.** 이 툴은 다양한 모터를 설계함 → 검증은 설계-무관 물리 불변량(자기일관성 e=-dλ/dt, dq 교차일치 T_FEM≈T_dq, 전류 선형성 T∝I, 3상 영합, 비선형 폴백 항등)으로 하고, 특정 형상의 절대값(airgap_B, emf_peak, load_T)은 "회귀 베이스라인"으로 분리. "정격 1.27N·m 대비" 같은 게이트는 금지. (왜: 샘플은 대용품일 뿐, 다른 설계 결과가 정상)
- **비선형 자기정자장 fixed-point는 완화계수 0.5에서 일부 극수(8극)가 한계진동**(iter7에서 안정→이후 1.66↔1.75 발산). relax=0.3 + RMS 수렴기준 + max_iter 40이면 8/10/14극 모두 5~15회 수렴. max|dB| 기준은 무릎 근처 소수 요소가 영원히 진동해 부적합 → RMS 사용. (왜: 설계마다 강성이 달라 댐핑·기준이 강건해야)
- **Maxwell 응력텐서 토크/공극 샘플링은 스테이터 내경 보어가 필요** → `exterior_xy()`는 외곽링만 줌(보어=내부링). `rings()`로 내부링 포함해야 r_gap 정확(아니면 r_gap이 요크로 가서 토크 폭발). (왜: shapely 차집합 형상의 구멍은 interior ring)
- **권선 phase 배정에 naive k%3 + 극성반전은 분수슬롯에서 위상 내 정확상쇄**(λ_A∝cos0−cos90+cos180−cos270=0). 권선함수(코일별 cos(α−축) 투영)로 대체 — 평형·전류주입 일관. 단 이는 lite 모델이라 일부 조합(8극12슬롯) balance~0.80로 저하 → 실제 설계는 slot-star 권선표 필요, Maxwell로 상별 검증. (왜: FSCW는 조합별 권선표가 정답)
- 수치 솔버 변경은 매번 feedback-runner(.feedback 하니스)로 검증: PASS(0)/OOT(2)/FAIL(1). venv 파이썬을 config `run`에 백슬래시 경로로(`.venv\Scripts\python.exe`) — forward-slash는 cmd가 거부. (왜: 시스템 py 3.14엔 sci 라이브러리 없음)
- **back-EMF 상평형 한계 근본원인**(실험으로 확정): 단일블롭 코일이 flux linkage를 한쪽 면(go side)의 mean Az로만 측정 → 위치별 DC 바이어스 → Phase A만 다르고 B=C(기하 x축 거울대칭). 메시밀도 무관(area 14→3 동일), 이산 최근접축 배정은 오히려 악화(10극 bal 0.71→0.32). cos-투영이 최선이나 기본파 bal ~0.63~0.71. **진짜 해결=go/return 도체쌍 권선 모델 + Maxwell 검증** → 검증 불가하므로 밤샘 자율작업에서 무책임하게 넣지 않음. peak bal은 10/14극 0.96, 8극 0.80. (왜: 검증 못 하는 물리 재설계는 금지; 근본원인+올바른 해법을 기록하고 보류가 정답)
