# Lessons

- 시스템 Python이 3.14(최신)면 과학 라이브러리 휠이 없을 수 있음 → 휠 호환 확실한 3.12 venv 사용. (왜: 빌드 환경부터 검증해야 "동작 안 함"을 피함)
- Korean 파일명을 Windows PowerShell 5.1에서 네이티브 exe(ffmpeg 등)에 넘기면 인코딩 깨짐("Illegal byte sequence") → ASCII 하드링크로 우회. (왜: WinPS 5.1이 레거시 코드페이지로 인자 전달)
- PowerShell에서 `$n번`처럼 변수 뒤에 한글이 붙으면 한글까지 변수명으로 파싱됨 → `${n}번`로 구분.
- "라이트 버전"=경량(lightweight)이지 밝은 테마가 아님. 밝은 색 UI가 사용자에게 "밝은버전?" 혼동을 줌 → 단계적 MVP/테마 의도를 명확히 알리고, 모호어는 초기에 확인. (왜: 진척·의도 오해 방지)
- GUI 검증은 포그라운드 전환(SetForegroundWindow)이 거부될 수 있으니 QWidget.grab() 또는 PrintWindow(flag=2)로 캡처. (왜: 다른 창이 겹쳐도 결정적 캡처)
- Windows 콘솔(cp949)에서 print에 em-dash(—) 등 유니코드 있으면 UnicodeEncodeError → 로그 출력은 ASCII로 하거나 파일로.
- "영상과 똑같이" 요청: ① 기능을 임의로 '라이트'로 빼지 말 것(영상에 있으면 구현 대상) ② 내 테마/스타일 입히지 말고 원본 외형(여기선 AEDT 라이트 회색) 따를 것. 두 실수가 "영상과 다르다"의 근본 원인. (왜: 사용자 핵심 의도는 '동일 재현'이지 내 디자인이 아님)
- 막연한 "아직 아니야" 피드백은 추측 패치로 이어져 드리프트. 각 수정은 **해당 실제 프레임을 직접 열어 그 화면을 근거로** 재현할 것(예: CreateCircle 대화상자 = uniq_v1/u_0031). (왜: 추측 대신 근거 기반이라야 수렴)
